Let’s get started

Download the plugin

QuantifiedDev starts with installing a plugin. The plugin is built for IntelliJ IDEA 11, 12 and 13 - both full and community editions. You will need java 1.6, 1.7 or 1.8. You can download the plugin here. You don’t need to unzip the folder, IntelliJ knows what to do with the zip. Once you have the plugin, open IntelliJ and go to your plugin settings. On windows and Ubuntu this is under the settings folder under the file menu. On OSX you go to preferences under the intellij menu. From the plugin settings click on the ‘Install plugin from disk button’. A file browser will appear, from here select the zip file with the plugin inside it. IntelliJ will install the plugin. IntelliJ will ask permission to restart, you should let it. After the restart you can go back to the plugin settings and see it installed.

Configure the plugin

QuantifiedDev attempts to guess where you are based on your IP. Often, this will default to a location that doesn’t truly reflect where you are. You can help the QuantifiedDev community by filling in your lat and long. Go to your settings, as described above, and go to QuantifiedDev settings. Here you will see your stream id, read token, latitude and longitude. More on stream id and read token later. Here you can edit lat and long with your correct location. Google search can help you find out your location.

Log your WTFs

Now that you have the plugin, whenever you encounter confusing, poorly written code you can log a WTF. Simply open the quantified dev tool window by going through the menu system: View > Tool Windows > QuantiedDev. Now, you should see the tool window with the WTF button. Go ahead and press it a few times.

View your WTFs

Now that you’re logging your WTFs you can go to quantifeddev.org/app and see your WTF history. First, open the quanitifed dev settings page. There is a handy button labelled ‘?' on the tool window to the left of the WTF button. Make a note of your stream id and the read token. Click on the button labelled ‘QD’ on the tool window and a browser will open at the QuantifiedDev dashboard. Now, enter your stream id and read token and press save. The graphs will update and you will be able to see your WTFs

Log builds

WTFs aren’t the only thing that you can log with QuantifiedDev. If you are working on a project that involves compiled languages you can also log builds. Just use the make command and we’ll send up a build event. Your build history is shown right alongside the WTFs.
