Detailed Instructions

1. Install the plugin

The plugin is built for IntelliJ IDEA 11, 12 and 13 - both full and community editions. You'll need java 1.6, 1.7 or 1.8. You can download the plugin here.
Don't unzip the downloaded zip file, IntelliJ knows what to do with the zip.

After downloading the plugin open IntelliJ and go to Plugin/Extension settings. On Windows and Ubuntu you'll need the Settings folder under the File menu. On OSX you go to Preferences under the IntelliJ menu. From the Plugin/Extension settings click on the ‘Install plugin from disk button’. A file browser will appear. From here, select the zip file with the plugin inside it. IntelliJ will install the plugin. IntelliJ will ask permission to restart, you should let it. After the restart you can go back to the plugin settings and see it installed.

2. Use the plugin

Whenever you encounter confusing, poorly written code hit the WTF button to log a WTF. Simply open the quantified dev tool window by going through the menu system: View > Tool Windows > QuantiedDev. Now, you should see the tool window with the WTF button. Go ahead and press it a few times. Dock the window for easy access.

3. View your WTFs

See your WTF history and other live WTFs happening around the world at quantifeddev.org/app. To get your personal data open the QuanitifedDev settings page from the "cog" button on the QuantifiedDev tool window. Copy your stream id and the read token and paste them into the form fields on the dashboard page and press "Save". The graphs will update and you will be able to see your WTFs. Click on "Community" to see other WTFs happening in the world.

Bonus! Log builds

WTFs aren’t the only thing that you can log with QuantifiedDev. If you are working on a project that involves compiled languages you can also log builds. Just use the make command and we’ll send up a build event. Your build history is shown right alongside the WTFs.

Other info

QuantifiedDev infers where you are based on your IP. Often, this will default to a location that doesn’t truly reflect where you are. You can override the default lat and long by filling in them in manually. Go to your settings, as described above, and go to QuantifiedDev settings. Here you will see your stream id, read token, latitude and longitude. More on stream id and read token later. Here you can edit lat and long with your correct location. Google search can help you find out your location.
